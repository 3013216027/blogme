第一周：输入/输出控制

by zhengdongjian2013@gmail.com

首先，对程序输入/输出的控制能力非常重要(Java中更甚，Java将流的概念利用到了极致!)。因此本周一~周五，就练习一下简单的输入/输出格式控制。这几题全部都是A+B Problem，题面也很好理解。希望能掌握基本的scanf/cin和printf/cout的控制姿势。(课上老师也许讲过诸如文件IO，EOF等，然而没有实践过或者应用过，几乎所有同学应该都是“哦，这样啊!我懂了！”的反应，然而却很难体会到其真切含义。。。）

Online Judge通过输入/输出的重定向来完成将文件流转化为标准输入/输出，在这里简单介绍一下。

就PC(Personal Computer)而言，屏幕显示就是标准输出，键盘的点按就是标准输入，对应地，显示器（或者叫监视器）就叫标准输出设备，键盘就叫标准输入设备。当然，打印机、扬声器都是输出设备，鼠标、触摸板等都是输入设备。

那么什么是输入/输出重定向呢？ 程序运行时要求的输入除了来自键盘，还可以来自其他设备，甚至包括文件！也就是说，程序可以接受来自键盘的数据外，还可以从文件里读取数据。输入重定向便是将来自其他文件的数据当做来自键盘输入的数据发给程序。stdio.h/cstdio中的函数freopen就可以完成这个功能(test1.cpp)：

#include <cstdio>
#include <iostream>
using namespace std;

int main()
{
    freopen("data.txt", "r", stdin);

    int a;
    cin >> a;
    cout << (a * 2) << endl;

    return 0;
}

其中同目录下还有一个文件data.txt，内容为：

28

在dev中打开test1.cpp，F11编译运行，可以发现，你并没有输入任何内容，但是程序却好像任然被输入了一个数28，并且在黑框中打印了结果28*2=56.似不似很神奇o_O

上面的程序将同目录下的文件data.txt中的内容当做键盘输入的数据发送给程序，于是程序后面的部分只需要通过cin/scanf来进行读取就可以了。

输出重定向也是一样一样的~~~

在OJ上，这一步是OJ帮你做了，因此你只需要写：

#include <iostream>
using namespace std;

int main()
{
    int a;
    cin >> a;
    cout << (a * 2) << endl;

    return 0;
}

OJ会自动帮你完成从一个文件中发送数据给你的程序，并且将你程序输出的结果拦截下来，传入另一个文件，最后只需要对比正确答案的文件和你程序生成的文件内容是否一致即可。
