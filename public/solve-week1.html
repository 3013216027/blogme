  • 1. HDU1089 A+B for Input-Output Practice (I)

题意：给若干组(a,b)，求a+b的结果。

题解：由于不知道有多少组，由此需要用到以下知识：用ASCII码保存的文本文件(本例中就是题目的输入数据文件)在被读取时会有一个问题，那就是如何判断是否读取到文件末尾？我们知道，ASCII码对应的数值范围是0~127，其中有0表示'\0'，48表示字符'0'，等等。但是每个ASCII码占用的是一个字节的内存，一个字节共有8位，按理说，应该可以保存从二进制编码00000000到11111111共256个字符才对！其实原因就是ASCII用的是带符号数，也就是8位二进制的第一位表示的是符号位，1表示负，0表示非负。于是数值范围其实是-128~127。-128~-1这一段是没有被占用的！于是C/C++中规定了一个特殊的短整型常量(const short)EOF，其值为-1，专门用来标记文件的末尾。也就是对一个如下内容的文件：

48

只有两个字符，而实际上却保存了3个字符，'4','8'和EOF。 （当然，你在windows系统下，变态的微软还会自作聪明地加上\r字符，会占用4字节，简直丧心病狂...) 那么EOF怎么在C++中用呢？看以下标程：

#include <iostream>
using namespace std;

int main() {
    int a, b;
    while (cin >> a >> b) {
        cout << (a + b) << endl;
    }
    return 0;
}

当然，也可以写成：

#include <iostream>
using namespace std;

int main() {
    int a, b;
    while (cin >> a) {
        cin >> b;
        cout << (a + b) << endl;
    }
    return 0;
}

这么写就可以了，具体原因似乎暂时还不好解释，因为这涉及到basic ios中的一个特殊的*类型转换函数*。总之，把cin >> a这种输入写到循环条件中，它就可以根据EOF自动判断是否读取到文件末尾啦~ 以后遇到的编程中，大多数情况你也不可能知道需要读取的文件的具体长度的，因此EOF是个很重要的概念。

  • 4. HDU1092 A+B for Input-Output Practice (IV)

题意给n个数，求和 每行第一个输入为n，遇到n=0时结束

#include <iostream>
using namespace std;

int main() {
    int n, a, sum;
    while (cin >> n) {
        if (n == 0) {
            break;
        }
        sum = 0; //init
        while (n--) {
            cin >> a;
            sum += a;
        }
        cout << sum << endl;
    }
    return 0;
}

  • 9. HDU1049 Climbing Worm

题意是一只虫子要爬出一个深度为n的洞（井？），每分钟可以往外爬u寸，但是爬一分钟就要休息一分钟，并且会下滑d寸，给出(n,u,d)，问你至少要几分钟爬出来。呃，虽然是很老的trick，还是要提醒一下，最后爬出后是不下滑的，比如(n,u,d)=(5,5,4)，只需要一分钟就可以爬出来啦！不要算成 ${{5} \over {5 - 4}}$ 哟~

简单思考一下，我们可以发现，以洞底/井底为坐标原点，设虫子最后一分钟的位置为x，那么一定有:

$$n \le x+u \lt n-(u-d)$$

移项，得:

$$n-u \le x \lt n+d-2*u$$

而虫子的坐标变换便是：

$$x' = 0 +u-d +u-d ... +u-d +u$$

其中

$$0 +u-d +u-d +u-d ... +u-d = x$$

于是便推出公式

$$ans = 1 + 2 * {\lceil{{n-u} \over {u-d}}\rceil}$$

其中『』表示向上取整。cmath中的ceil函数可以向上取整，但是其传入参数和返回值都是浮点数，永远记住：在计算机中，浮点数是绝对不精确的！对于整数，这里有一个小技巧，m除以n向上取整在C艹中可以写成：

$${m+n-1} \over {n}$$

不难验证上述的正确性。于是

$$ans = 1 + 2 * {\lceil{{n-u} \over {u-d}}\rceil} = 1 + 2 * \left({{n-d-1} \over {u-d}}\right)$$ 


#include <iostream>
using namespace std;

int main() {
    int n, u, d;
    while (cin >> n >> u >> d) {
        if (n == 0) {
            break;
        } else {
	    //Update[2015-06-01]: 先除以u-d向上取整后再乘2，
	    //之前用LaTex写公式，因此没注意要加个括号...
            cout << (1 + (n - d - 1) / (u - d) * 2) << endl;
        }
    }
    return 0;
}
